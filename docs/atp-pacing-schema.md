# ATP Pacing Schema (`data/atp-pacing.json`)

**Status:** DRAFT — for review before extractor automation runs.
**Goal:** 100% lossless capture of the DBE Annual Teaching Plan PDFs at the
repository root, so the worksheet/test generator and future planner /
lesson / weekly-plan features can read everything from one structured file
instead of going back to PDFs.

The PDFs are the **single source of truth**. This file is a derived
artefact: regenerated by `scripts/extract-atp-pacing.js`, never hand-edited.
Every record carries a `source.pdf` + `source.page` pointer so any field can
be audited back to the page it came from.

---

## Top-level shape

```jsonc
{
  "$schemaVersion": "1.0.0",
  "extractedAt": "2026-05-02T14:30:00Z",
  "extractor": { "model": "claude-sonnet-4-6", "promptVersion": 1 },
  "subjects": {
    "<Subject Name>": {            // e.g. "Mathematics"
      "<grade>": {                 // "4" | "5" | "6" | "7"
        "<year>": { /* PacingDoc */ }   // "2026" | "2023-24"
      }
    }
  }
}
```

Subjects use the same canonical names as `data/atp.json` so the existing
`SUBJECTS` list keeps working (e.g. `"Mathematics"`, `"Natural Sciences and
Technology"`, `"Social Sciences — History"`). Year is explicit so the 2026
PDFs and the 2023-24 PDFs can both be present and the consumer picks.

## `PacingDoc` (per subject × grade × year)

```jsonc

  "subject": "Mathematics",
  "grade": 4,
  "year": "2026",

  "language": "English",          // for HL/FAL ATPs we record the LoLT here
  "source": {
    "pdf": "2026_ATP_Mathematics_Grade 4.pdf",
    "totalPages": 8,
    "publishedBy": "DBE",
    "documentTitle": "2026 ANNUAL TEACHING PLANS: ENGLISH MATHEMATICS: GRADE 4"
  },
  "capsStrands": [                // high-level CAPS content areas, verbatim
    "Numbers, Operations and Relationships",
    "Patterns, Functions and Algebra",
    "Space and Shape (Geometry)",
    "Measurement",
    "Data Handling"
  ],
  "terms": {
    "1": { /* TermPlan */ },
    "2": { /* TermPlan */ },
    "3": { /* TermPlan */ },
    "4": { /* TermPlan */ }
  },
  "yearOverview": {                // optional — only when PDF has a summary
    "totalWeeks": 41,
```

## `Unit` — the atomic teaching block

A "unit" is one topic-block as it appears on the ATP grid: it spans one or
more weeks and has its own concept tree, prerequisites, and (sometimes) its
own resources/informal assessment. Every PDF row maps to one or more units.

```jsonc
{
  "id": "math-4-2026-t1-u3",                // stable, regeneratable
  "ordinal": 3,                              // teaching order within the term
  "weeks": [4, 5, 6],                        // all weeks the unit spans
  "hours": 18,                               // total hours allocated
  "durationLabel": "2½ weeks",               // verbatim if PDF used a label
  "topic": "Whole numbers: Addition and subtraction",
  "capsStrand": "Numbers, Operations and Relationships",
  "subStrand": null,                         // when CAPS uses a finer label
  "isRevision": false,
  "isAssessmentSlot": false,                 // true for "FORMAL ASSESSMENT TASK" columns
  "subtopics": [ /* Subtopic[] — the full concept tree */ ],
  "prerequisites": [ /* string[] — verbatim "prior knowledge" bullets */ ],
  "resources": [ /* string[] — verbatim "Resources to enhance learning" */ ],
  "informalAssessment": [ /* string[] — verbatim formative ideas */ ],
  "notes": [ /* string[] — any "Note: ..." callouts attached to this unit */ ],
  "source": { "pdf": "2026_ATP_Mathematics_Grade 4.pdf", "page": 1 }
}
```

### `Subtopic` (recursive)

The concept-and-skills column in a CAPS ATP is a hierarchy: heading →
bullets → (sometimes) sub-bullets. We preserve all three levels.

```jsonc
{
  "heading": "Number range for calculations",
  "concepts": [
    "Addition and subtraction of whole numbers of at least 4-digits"
  ],
  "children": [
    {
      "heading": "Calculation techniques",
      "concepts": [
        "Use any two techniques to perform and check written and mental calculations of whole numbers including:",
        "— Estimation",
        "— Building up and breaking down numbers",
        "— Using a number line",
        "— Rounding off and compensating",
        "— Using addition and subtraction as inverse operations"
      ],
      "children": []
    }
  ]
}
```

Bullets that are clearly sub-bullets in the PDF (em-dashes, indented hyphens,
ASCII bullets) are flattened into the parent's `concepts[]` array with their
leading marker preserved, so a renderer can decide whether to render them as
a nested list or inline.

## `FormalAssessment`

```jsonc
{
  "label": "Assignment",                  // verbatim from PDF
  "type": "Assignment",                   // canonical: Assignment | Investigation | Project | Test | Practical Task | Examination
  "term": 1,
  "weeks": [4],                           // when on the timeline
  "minMarks": 25,                         // standard CAPS range
  "maxMarks": 50,
  "weightingPercent": null,               // some ATPs specify
  "topicsCovered": [
    "Whole numbers: Counting, ordering, representing",
    "Number sentences",
    "Whole numbers: Addition and subtraction"
  ],
  "conditions": "To be completed in class within three hours",
  "rubricNotes": [],                      // any rubric/marking-guide hints in the PDF
  "source": { "pdf": "2026_ATP_Mathematics_Grade 4.pdf", "page": 1 }
}
```

## How NS & T (and other "duration"-style PDFs) map in

The 2026 NS&T PDFs use a `Duration | CAPS Topics | Core Concepts | Resources |
Informal Assessment` row layout instead of the week-grid. They map cleanly:

- `Duration` → `Unit.durationLabel` (verbatim) + `Unit.weeks` (computed from
  cumulative duration within the term)
- `CAPS Topics` → `Unit.topic`
- The strand heading on the page (e.g. "Life and living") → `Unit.capsStrand`
- `Core Concepts, Skills and Values` → `Unit.subtopics`
- `Resources to Enhance Learning` → `Unit.resources`
- `Informal Assessment` → `Unit.informalAssessment`
- The "SBA (FORMAL ASSESSMENT)" footer → `TermPlan.formalAssessment`

For Languages ATPs (English/Afrikaans HL/FAL), the column structure is
different again — usually `Listening & Speaking | Reading & Viewing | Writing
& Presenting | Language Structures & Conventions`. Those become four parallel
`Unit`s per week, each carrying its own `capsStrand` (= the language skill).

## Generator migration (after extraction)

`api/generate.js` currently consumes `ATP[subject][grade][term].topics` (a
flat string list). After migration:

1. `lib/atp.js` will load **both** `atp.json` (legacy index) and
   `atp-pacing.json` (rich data), and expose a new `getPacingUnit(subject,
   grade, term, topic)` that returns the matching `Unit`.
2. `buildContext()` will pull the unit's `subtopics`, `prerequisites`, and
   `informalAssessment` into the prompt so generated questions can reference
   the actual concepts-and-skills from the ATP rather than improvise.
3. `data/atp.json` will be auto-derived from `atp-pacing.json` (a build step)
   so the two can never drift. The flat topic list it currently exposes
   becomes a projection, not a hand-written file.

## Resolved decisions

1. **Year selection — use the latest available.** Generator picks the most
   recent year for which a `PacingDoc` exists. So 2026 wins for Maths Gr 4-5
   and NS&T Gr 4-7; 2023-24 is used for everything else until newer PDFs land.
   Both years remain in `atp-pacing.json` so we can still serve historical
   resources or A/B compare.
2. **Languages PDFs — parallel `Unit`s per week.** Each weekly row in an
   English/Afrikaans HL/FAL ATP becomes up to four parallel `Unit`s, one per
   CAPS skill, distinguished by `Unit.capsStrand`:
   - `"Listening and Speaking"`
   - `"Reading and Viewing"`
   - `"Writing and Presenting"`
   - `"Language Structures and Conventions"`
   All four share the same `weeks` value but each carries its own subtopic
   tree. This matches the planner's per-skill column view and lets the
   worksheet generator scope to a single skill cleanly.
3. **Mark allocation — capture per-section splits.** `FormalAssessment`
   gains a `sections[]` array. Each entry records the section label, mark
   value, and (where the PDF specifies) the skill or topic it tests.

### Updated `FormalAssessment` shape

```jsonc
{
  "label": "Formal Assessment Task 1: Oral",
  "type": "Oral",
  "term": 1,
  "weeks": [1, 2, 3, 4],
  "minMarks": null,
  "maxMarks": 20,
  "totalMarks": 20,
  "weightingPercent": null,
  "sections": [
    {
      "label": "Read Aloud",
      "marks": 20,
      "capsStrand": "Reading and Viewing",
      "notes": "Commence with this task in Term 1 and conclude in Term 2 when the mark will be recorded."
    }
  ],
  "topicsCovered": [],
  "conditions": null,
  "rubricNotes": [],
  "source": { "pdf": "1.050 ATP 2023-24 Gr 4 Eng HL final.pdf", "page": 1 }
}
```

When a PDF only gives a mark range (e.g. CAPS Maths "25-50 marks"),
`sections` stays empty and `minMarks`/`maxMarks` carry the range.
